## Decisions

- The current playable branch should not look like a disconnected event card. It belongs to the same age-based life timeline as the early-life entries.
- Action resolution should not become its own ordinary-player event card. It is part of the consequence of the event the player just acted on.
- If an event's choices mention a person, object, place, clue, faction, or item, the event body must first establish why it is present and what the player currently knows.
- `fuzzySuccessLabel` is treated as risk/difficulty display, not as a place to summarize the chosen action or declare success.
- Fate preview is a confirmation dossier only. Early life, hidden hooks, suspicious objects, hidden NPC functions, and unexplained details stay in `opening.earlyLifeTimeline`, backend hidden state, or GM/debug until discovered.
- The ordinary fate preview body must be built by the engine from structured run data, never by filtering AI provider prose. Text sanitizers are leak-prone because the provider can hide spoilers inside allowed fields (e.g. NPC names inside 出生家庭) or use unexpected labels (e.g. 出生时间). Deterministic engine construction is the authoritative spoiler-safe path; sanitization is only a fallback.
- The ordinary fate page shows five fields only (出生地点、出生家庭、家境表现、命运预览、当前处境). This is a compliant subset of the GPT.md allowed dossier fields, chosen for a clean "who am I" confirmation rather than a stacked block of setting prose.
- The engine, not the provider, owns the first-action age. The provider can disobey the prompt and return a tiny `timeSpan.ageEnd`, so the opening age must be clamped server-side before it is applied to the run. Trusting provider age caused infant-aged playable branches with over-aged choices.
- Ordinary playable events must never surface AI-failure or generation-rule text. Rather than fabricating clean filler (which still shows generic content and masks failure), a degenerate provider body should fail validation and route to the age-aware mock generator, which produces real age-appropriate content. Fabrication is removed entirely; debug/meta wording belongs only in GM/debug.
