## Opening-Sequence Rework (Second Session)

### Completed

- Reworked the run opening flow so a new life no longer starts at a playable event. New flow: setup -> AI/engine generates a non-interactive opening sequence (birth background card + fate preview + early-years auto progression) -> player clicks `开始人生` -> first meaningful branch with 3 choices + optional free-form entry.
- Added `src/opening-sequence.js`: generates the opening `life_event` with `interactionMode: non_interactive`, 0 choices, `freeform.allowed: false`, a Chinese birth-background card (birth place, family, guardians, family-background expression, talent first manifestation, outside reaction, world context, initial NPCs, initial situation, foreshadowing), a world-specific early-years narrative, early manifestation-ratio state patches, and `opening.phase = first_branch_ready`.
- First action age: default 7; 5 when early-consciousness talents (reincarnation memory, fate/destiny/mythic/dream-walker patterns) or very high Intelligence potential justify it. Exported `firstActionAge` for provider prompts.
- Extended `src/play-session.js` with an opening phase: `createPlaySession`/`createPlaySessionAsync` return `openingPhase: "background"` + `inputRequired: "opening_continue"` for new runs; choices and free-form input are blocked (no turn consumed) until `advance_opening`; added `start/begin/开始/开始人生/继续人生` input normalization; `advanceOpeningSync/Async` generate the first branch event and handle immediate endings; ended sessions ignore further inputs.
- Loaded saves with event history skip the opening and continue at a playable event as before.
- Extended the AI provider layer: mock provider uses the local opening generator; DeepSeek/OpenAI-compatible providers get `generateOpeningSequence` with `buildOpeningSequencePrompt` (opening rules, firstActionAge, required body fields, must-not list, run/world snapshots) and a dedicated `non_interactive_opening` system-prompt branch (0 choices, no freeform, fate-preview title, no empty yearly diary, no fixed plot, opening.phase patch). Provider failure falls back to the local opening with `provider_fallback` flags.
- Extended `src/web-session-store.js`: serialized `openingPhase`, normalized `advance_opening` action; GM dev-scenario injection now clears `openingPhase` so scenario testing can bypass the preview page.
- Reworked `web/index.html` + `web/app.js`: separate fate-preview panel (talents, attributes, background story, initial relationships, `开始人生` button) shown during the opening phase; ordinary event card hidden until the first branch; loading indicator (`命运正在描绘中……`) appears immediately on any action with all interactive controls disabled; failure shows `命运暂时迷失，请重试`; localized labels extended (cultivation attribute display names 仙姿/悟性/根骨/出身/底蕴/气运, progress labels, progress value labels, talent name labels, risk labels) so ordinary players see Chinese instead of backend IDs.
- CLI `npm run play`: opening phase prints the fate preview and waits for `开始`/`start`; scripted mode auto-advances through the opening; new hint text for `opening_continue`; default `--ending-age` raised 6 -> 12 because the opening already advances to age 5-7.
- Updated `tools/check-playtest-readiness.mjs`: now verifies the opening phase exists, is non-interactive, blocks premature choices, auto-advances early years, and only then exposes 3 choices + free-form at the first branch; ending check goes through `开始人生`.
- Updated `tools/smoke-web-playtest.mjs`: asserts the started session is an opening (background phase, 0 choices, freeform false, age >= 5), advances with `advance_opening`, then exercises the playable flow; default ending age 5 -> 12; ending loop turn cap raised.
- Updated `tests/play-session.test.mjs` (opening assertions, blocked inputs, advance flow, adjusted history counts) and `tests/web-session-store.test.mjs` (opening serialization, full preview->first-branch walk, blocked choice before advance).
- Documented the opening-sequence contract in `docs/ai-output-protocol.md` (new Opening Sequence Requirement + Action Resolution Continuity sections) and `docs/mvp-program-skeleton.md` (new Opening Sequence Rules section, default ending age 12).

### Decisions

- The opening sequence is one non-interactive `life_event`, not a new responseType; it reuses the existing validator (`non_interactive` + 0 choices is already legal).
- Opening auto-progression must not be an empty yearly diary and must not force a major event every year; it is life undertone + family + talent manifestation + world rules + foreshadowing.
- Free-form input opens only at the first meaningful branch. During the opening phase, choice/freeform inputs are rejected with `inputRequired: "opening_continue"` and consume no turns.
- `继续` stays reserved for freeform confirmation; the opening advances via `开始`/`开始人生`/`继续人生`/`start`/`begin`/`continue`.
- Early manifestation patch: by first action age, manifested values reach roughly 24%-34% of potential (Family Background manifests fully), aligned with the manifestation-ratio rules.
- Cultivation attribute display names map 1:1 onto the five base attributes; no new base attributes are introduced.
