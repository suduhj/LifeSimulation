## Decisions

- AI prose is no longer treated as the authority for continuity-critical story facts. The engine records closed facts and thread stage in `worldState.storyState`; AI and mock providers render within the next event contract.
- The MVP implementation stays intentionally narrow: it protects the cultivation jade-talisman/lingyinfu repeated-discovery loop first, while leaving broader thread extraction for future expansion.
- Provider failures should keep the playtest session alive with marked local fallback, matching the existing README/technical-standard promise. Fallback events must still honor story contracts.
- Repeated yearly scene structures are a separate class of continuity failure from repeated facts. They should be handled by an annual fact package and forbidden event-shape validation, not by hardcoding cooldowns for one named plotline.
- The annual fact package is now the default authority for what a cross-year branch is about. A long-running thread may continue, but it cannot consume every year as the main event unless the Year Tick selects it as the primary life delta.
- The MVP solution for repeated annual scenes is the general five-axis director, not a hardcoded cooldown for "后山" or any other single plotline. Continuing threads can still matter, but the main yearly event must come from structured state pressure and be written back as engine-owned facts.
- Attribute growth follows the same state-first principle as story continuity. AI can write how growth feels, but the authoritative current ability comes from the engine-owned Growth Ledger, not from prose or direct AI-authored numbers.
- Talent bonuses enter potential immediately, but age maturity caps and realized evidence decide current effective power. Reaching adulthood removes the hard age cap; it does not automatically cash out all locked potential.
- Ordinary UI should explain attributes as current / realized / potential / locked / attention values. Raw backend terms such as `effective`, `realized`, and `growthLedger` stay in developer/debug surfaces.
