# 返工 Sprint 计划（评审链共享文档）

> 这份文档由"团队评审链"逐站填充：产品(office-hours) → CEO → 工程 → 设计 → 高工(review) → 测试(qa)。
> 每站只在自己的小节追加，最终汇总成带验证方式的 P0 清单。本阶段不改代码。

## 已确认需求（用户拍板）

- **R2 属性显化 = 按年龄显化**：潜能 = 基础加点 + 天赋加成；加点页/命运页实时显示"基础+天赋=潜能"；当前表现随年龄逐步逼近潜能，约 18 岁基本全显化。天赋与玩家加点同一套规则。
- **R3 AI 失败兜底 = 先重试，再上下文兜底**：DeepSeek 失败/修复后仍泄露 → 先自动重试数次；仍失败 → 上下文感知兜底（引用最近人物/线索、不矛盾、不凭空换场景）。
- **产品主轴 = 真 AI + 零接缝**：核心卖点是"涌现式 AI 人生"，重点投入 AI 可靠性/连续性，把 mock 兜底降为"几乎看不见"的底。

---

## 第 1 站 · 产品经理（office-hours）

**魔法时刻**：`自由行动 → AI 合理判定 → 世界变化 → 人生持续推进`——玩家要的"哇"是"我的选择真的被世界记住并改变了后面的人生"。

**核心产品判断**：不缺内容，缺的是"永远不让玩家看到接缝"。每一次接缝外露（mock 突兀、泄露、年龄跳错）都直接破功。选了最难的路（真 AI + 零接缝），但做成了就是固定剧本版给不了的体验。

**产品优先级**：
- **P0 · 消灭接缝**：mock 兜底不再突兀（R3）；修年龄不一致、未行动预显示"+1"等失信细节；泄露护栏（已基本完成）。——决定"产品成不成立"。
- **P1 · 让成长可见**：属性按年龄显化 + 加点页实时显示（R2）。——决定"玩家愿不愿意继续投入"。
- **P2 · 深度与变量**：更多世界/天赋/结局分支。——接缝补好前不碰，否则内容越多越易露馅。

**Assignment（产品作业）**：下一次真机试玩时，只盯一件事——记录"哪一刻你感觉这不是真人生，而是程序在跑"。每一个那样的瞬间就是一个接缝，进 P0 候选。

## 第 2 站 · CEO / 创始人（plan-ceo-review）

**唯一要做对的一件事**：核心循环**不破功**。其余全是噪音。

- **范围纪律**：P2（更多世界/天赋/结局）这轮**一律不碰**。在接缝补好前加内容 = 给更多地方露馅。任何"顺手加个功能"的冲动直接砍。
- **成功指标（可量化）**：给"零接缝"一个硬标准，否则永远说不清"做好了没"。建议：**真机连跑一条完整人生（0→结局），玩家可见文本里 0 处英文/ID/后台词、0 处年龄跳错、mock 兜底触发率 < 10%**。达不到就不算过。
- **最大战略风险**：真 AI 本质不稳定。"零接缝"不是"AI 永不失败"，而是"失败了玩家也看不出来"。所以 R3（重试+上下文兜底）不是优化，是**产品命脉**——它决定核心卖点成不成立。
- **CEO 裁决**：P0 = 一切让玩家"看出这是程序"的接缝；P1 = 成长可见（留住玩家）；P2 = 砍到补完接缝之后。先把"一条不破功的人生"做出来，再谈广度。

## 第 3 站 · 工程经理（plan-eng-review）

**架构判断**：现有分层（引擎校验/保存 + AI 提案 + mock 兜底）是对的，不重写。问题集中在三处接缝点。

- **R3 重试+上下文兜底**（命脉）：
  - 在 `safeGenerateLifeEvent`/`safeGenerateActionResolution`/`safeGenerateOpeningSequence`（[play-session.js](src/play-session.js)）加**有上限的重试**（建议 2 次，失败原因含校验/泄露），仍失败才兜底。
  - 兜底改为**上下文感知**：把最近 importantNPCs / 未结线索传入 `buildMockBody`（[mock-ai.js](src/mock-ai.js)），引用已有人物（如"银团"）、不凭空换场景。
  - 风险：DeepSeek 慢，重试叠加延迟 → 必须设超时与重试上限；前端 loading 要交代"仍在生成"。
- **R2 按年龄显化**：
  - 模型：`manifested = clamp(max(已有值, floor(potential * ageCurve(age))), 0, potential)`；`ageCurve(0)=小起点`，`ageCurve(18)=1.0`。
  - 落点：出生（[initial-run.js](src/initial-run.js)）、早年（[opening-sequence.js](src/opening-sequence.js)）、每个事件推进（[run-loop.js](src/run-loop.js) applyStatePatch 后做一次"按龄显化"pass）。
  - 必须与 AI 的 manifestationChanges **协调**：年龄曲线是"下限"，AI/事件在其上叠加，统一封顶=潜能。**严防重复计入**（天赋加成在 init 已进 potential，年龄曲线只动 manifested）。
  - 显示：setup 加点页（[web/app.js](web/app.js) / [web/index.html](web/index.html)）实时显示"基础+天赋=潜能"；当前命运页/状态摘要已有三层显示，复用。
- **失信细节 bug**：
  - 年龄不一致：根因在 `buildMockBody` 写死 `nextAge` 正文，与 `normalizeFirstBranchEvent` 强制的 timeSpan/标题年龄不同步。修法：正文不写死年龄，或正文年龄统一取 `timeSpan.ageEnd`（归一化之后）。
  - 未行动 +1：mock 的 life_event 自带 `visibleChanges` 进度+1。修法：fresh life_event 不携带/不渲染状态增益，增益只在 resolution 出现。
- **排序（小步、可独立验证）**：① 修年龄/+1/空格（便宜、低风险）→ ② R3 重试 → ③ R3 上下文兜底 → ④ R2 显化+显示。

## 第 4 站 · 设计师（plan-design-review）

- **兜底必须"隐形"**：mock 事件在视觉/文风上要和 AI 内容**无法区分**——同样的标题格式、同样的结算结构、引用同样的人物。玩家永远不该感到"这一段是降级的"。
- **成长可见（R2 的体验）**：加点页用"基础 4 +天赋 +25 = 潜能 29"这种**可读公式**，别只甩一个总数。状态摘要里"当前 3 / 潜能 34"要让玩家直觉到"还会涨"——可加一条细进度条或"显化中"提示，但别盖过叙事（遵循 design-standards：故事优先，数值不抢戏）。
- **年龄一致性是信任问题**：时间线标签、标题、正文三处年龄必须永远一致；不一致 = 系统在你眼前出错。
- **结算只在行动后出现**：新事件卡上不出现任何"+X"；"+X"只出现在已结算的时间线节点里（design-standards 已有此规则，按它执行）。
- **重试中的 loading**：`命运正在描绘中…` 在重试时不能让人以为卡死；可在超过 ~3s 时补一句"正在仔细推演这一年"。

## 第 5 站 · 高级工程师（/review，针对现状+计划的代码级风险）

- **R3 重试**：必须有硬上限 + 超时，禁止无限循环放大成本；重试要带"上一轮失败原因"喂回修复提示，提高一次成功率。加测试：失败1次→重试成功不出 mock；恒失败→兜底且带 `provider_fallback` 标记。
- **R2 双重计入风险**（最危险）：天赋加成在 `createInitialRun` 已计入 `potential`；年龄曲线**只能改 manifested**，绝不能再加一次到 potential。否则属性随年龄虚高。加测试：固定 potential，跨年龄断言 manifested 单调逼近且永不超过 potential。
- **兜底上下文化**：把 NPC/线索传进 mock 时，注意**别把隐藏 NPC 的 hiddenInfo/真实身份泄露**进 mock 正文——复用现有 `visibleNpcLabel`/泄露护栏。
- **年龄归一化**：修 `buildMockBody` 时确认所有 mock 正文（三世界）都不再写死年龄；加测试断言 `timeSpan.ageEnd` 与正文/标题/时间线年龄一致。
- **回归**：所有改动后 `npm test` + `validate:data` + `smoke:web` 必须全绿。

## 第 6 站 · 测试（/qa，验收场景）

- **接缝总扫**（真机，DeepSeek）：连跑一条完整人生，逐屏记录英文/ID/后台词、年龄跳错、突兀兜底；目标 0 接缝、mock<10%。
- **R3**：注入"失败1次"的假 provider → 断言重试后用 DeepSeek、无 mock；注入"恒失败" → 断言兜底事件引用了最近 NPC（如银团）、未引入矛盾新物、带 fallback 标记。
- **R2**：跨年龄多事件跑 → 断言 manifested 随年龄上升、~18 岁≈potential、不超 potential；setup 预览断言显示"基础+天赋=潜能"。
- **失信细节**：断言任一事件三处年龄一致；断言 fresh life_event 玩家可见无"+X"，"+X"只在已结算节点。
